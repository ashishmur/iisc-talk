## The One-Person Billion-Dollar Startup & Organizational Design

We're covering in some part of our segment around the one-person billion-dollar startup. It's basically — you're able to understand organizational structure. OB (organizational behavior) and organizational structure becomes such an important, interesting concept and topic. You need to have that perspective when you're thinking about how you build AI. You see, the AI has some similar challenges to a human being — you are just smart enough, you can be a little inconsistent, you can't do too much, you have to do a little bit and you can run a little bit really fast, then go to the next task.

**One of the best ways to design AI systems is to replicate the same processes and structures which organizations have already incorporated.** They spent the last century optimizing the right way units of work or units of labor can move inside an organization to ultimately create value.

Take a very simple example — product management. The key job I have is to build a spec document, a PRD (product requirement document). The PRD covers strategy — "why are we doing it?" There's usually a strategic paper, a vision document for your product. Then you get into the detailed document: you do user research. Based on the user research, you understand user problems, prioritize certain problems, think about solutions. Then write a PRD which is nothing but a set of product experience instructions you're giving to engineers and designers. Designers create the design, give to engineers. Engineers read the PRD, build a technical design document, which in turn gets broken up into tasks on a Jira board which individual engineers start executing on.

There's a lot of coordination here — engineers work on the same codebase and have always been using something called Git, which allows independent engineers to complement each other's work.

**And so when you think about an AI system, you'd say: why don't we break the process or the flow of the AI down across each of these stages?** It has limited context. Can't do everything at the same time. We can break it down in chunks and get it done. So we first get the AI to do market research. We then get it to build a strategy document. You get it to go through user research, create surveys if you want, analyze the data from the surveys using AI. Then create a technical requirement document, which in turn becomes individualized features. And then you get each individual agent to go and build each part of that system out. And what you end up building is far more superior, far more complex, with far better taste, and a slightly higher likelihood of actually achieving its outcomes.

When engineers do their work, how do you know if they've done a good job? And if they haven't broken something? They write unit tests. I said, okay — we should always write unit tests. That process works for human beings. It's got challenges because human beings are not perfect. AI is not perfect too. It could hallucinate, it could introduce a bug, it could accidentally break the codebase. The same challenges exist. These challenges are not new. We've always been dealing with them. And we're continuously building better processes. Now you're only going to be doing that with agents.

**But your ability to understand the process, understand and replace labor with compute, and say "I understand how compute can create value in the form of a workflow"** — that allows me to take simple intelligence and turn that into value. Creating revenue for the organization or sustaining and increasing revenue.

## Building Context & Becoming Generalists

An important thing to be able to do is to be **building context.** The more context you build, the more you have the ability to have the AI be very nuanced in the way it's working in different scenarios. As individuals in teams start identifying what the right set of workflows are, they should start documenting that and have the context graphs for each of those created.

Over time, we'll start figuring out how combined context graphs across multiple individual workers start coming together — that creates organizational context. Which now allows slightly more abstracted leaders to start thinking: "I have the ability to get outcomes from marketing. I have the ability to get outcomes from engineering, different functions, different regions." Individual folks have already figured out and created the right context in terms of how an AI has to solve problems over here. That sets the foundation.

**One thing we're definitely going to see is we're going to become a lot more generalists in nature.** I already need to know how to be a very good designer right now. The expectation is I'm already bringing in really well-designed visual prototypes and working code. I need to be able to build systems and applications. I cannot say no anymore. That expectation has fundamentally changed. Everybody is prototyping something. In fact, leadership believes if you cannot prototype it, that idea cannot be built. As simple as that.

And so we are really building agents ourselves and saying, "I think there's a better way to get an agent to do financial analysis. And it involves giving it access to some of these tools and instructions." This makes it a lot more superior. You hand that out to engineers — they already have the oversight of the idea and can quickly scaffold it and include it as part of the platform.

**It's definitely the age of the polymath.** I believe there's a Renaissance of thinkers which we're going to start seeing. Folks who are able to work across really diverse contexts because they figured out how the fundamental unit of labor at the lower levels can be achieved, and then they have workflows and systems which allow these agentic entities to deliver output in these areas, and they are now coordinating across diverse problem sets.

You could be looking at finance, marketing, growth — all at the same time. We'll start having to develop the ability to look at the interconnectedness of problems. Today's talk is in that direction — we've covered a broad set of topics and themes. But that's how all of us are going to be. We're going to be absolute generalists. With the ability to now allow a bunch of really diverse ideas to snowball into a lot of innovation.

There's a belief that we're moving into the **age of abundance.** And if that's the case — zoom in for a moment and assume it could be true — it's a brilliant opportunity for all of us to be able to create. And to be able to solve our way through, build a muscle, have our own systems, be able to work across diverse ranges.
